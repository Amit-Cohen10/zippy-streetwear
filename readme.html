Technical Implementation

What was hard to implement:

  • Cookie-based Authentication: Implementing secure JWT tokens with HTTP-only cookies and proper session management
  • Data Persistence: Creating a robust file-based storage system that maintains data across server restarts
  • Exchange System: Building a complex trading system with user matching and item validation
  • DOS Protection: Implementing rate limiting and security measures to prevent attacks
  • Real-time Updates: Ensuring cart and inventory updates are reflected immediately across sessions

Data Persistence

All user data is persisted in JSON files in the file system:

  • User Details: Stored in data/users.json
  • Cart Data: Stored in data/carts.json
  • Purchase History: Stored in data/orders.json
  • Login Activity: Stored in data/activity.json
  • Products: Stored in data/products.json
  • Exchanges: Stored in data/exchanges.json

Data persists across server restarts and is loaded on startup.

UI Customization

The platform supports dark/light theme customization using localStorage. Users can toggle between themes, and their preference is saved and restored on subsequent visits.
